# 项目通用设置

## 里程碑归档规范

每次功能完成/变更，必须在 `MILESTONES.md` 中更新对应阶段的完成情况和勾选清单。`MILESTONES.md` 是项目进度的唯一真相源。

更新时机：

- **阶段开始时**：将状态从 ⬜ 改为 🔄，填入开始日期
- **每个任务完成时**：勾选对应的 `[x]`
- **阶段完成时**：将状态从 🔄 改为 ✅，填入完成日期，更新总览表
- **有新发现/阻塞时**：在阶段下追加备注

阶段状态图例：

- ⬜ 未开始
- 🔄 进行中
- ✅ 已完成
- ⛔ 已阻塞

## 变更日志规范

每次功能改进/变更，必须在 `CHANGELOG.md` 中记录。格式约定：

- 以 `## [版本号] - YYYY-MM-DD` 为版本标题
- 按类型分组：`### Added` / `### Changed` / `### Fixed` / `### Removed`
- 每条记录以 `-` 开头，简洁描述变更内容和原因
- 版本号与 `MILESTONES.md` 中的里程碑对应

示例：

```markdown
## [2.1.0] - 2026-07-03

### Added
- 岗位绑定功能：会话创建时可关联 JobPosition

### Changed
- 认证方式从单一令牌改为双令牌（admin_token + candidate_token）

### Fixed
- 修复 WebSocket 断线重连后对话历史丢失的问题
```
